# Coding and Code Base Architectural Guidelines
- Modularization is essential for maintainability and scalability.
- High depth with simple interfaces is a good practice.
- Low decoupling for maintainability and scalability.
- The goal is to keep the code base as clean and simple as possible with efficient codes, no over-engineering or things to prove, just practical, efficient, easy to maintain codes.
- Code base architecture should be modularized to bits with low decoupling so that even for a big code base, LLMs would not need to read the whole monolithic code base just to add a feature or fix a bug, locality of code is important.
